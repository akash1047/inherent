# Support

Use the right channel for the right kind of request.

## GitHub Issues

Use issues for:

- reproducible bugs
- narrowly scoped feature requests
- documentation mistakes

## Pull Requests

Open a PR when you already have a concrete fix or improvement.

## Security Reports

Use [SECURITY.md](https://github.com/inherent-prime/inherent/blob/main/SECURITY.md). Do not report vulnerabilities in public issues.

## Maintainer Contact

If you need direct maintainer help for repository-level questions, contact the repository owner listed in GitHub.
