# Security Policy

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Report security issues privately to the maintainers through the security contact for this project. If a dedicated security contact is not yet published in the repository settings, contact the repository owner directly and include:

- affected component
- impact
- reproduction steps
- suggested remediation if known

## What to Expect

- Acknowledgement after the report is reviewed
- Triage and reproduction by a maintainer
- A fix and disclosure plan based on severity and exploitability

## Scope

This policy covers code and configuration stored in this repository. It does not cover private systems or deployments that are not part of the OSS repository.
